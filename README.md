# LowCode System

Система low-code для создания интерфейсов через YAML конфигурацию.

## Описание

Система позволяет создавать веб-интерфейсы, описывая их структуру и поведение в YAML файлах. Поддерживаются различные типы виджетов, динамическая валидация, и выполнение команд.

## Установка и запуск

1. Установите зависимости:
   ```bash
   python3 -m pip install -r requirements.txt
   ```

2. Запустите приложение:
   ```bash
   python3 app.py
   ```

3. Откройте в браузере:
   - Главная страница: http://localhost:8000/
   - Список страниц: http://localhost:8000/pages
   - Панель отладки: http://localhost:8000/debug

## Создание новых страниц

1. Создайте папку в директории `pages/` с именем страницы
2. Добавьте файл `attrs.yaml` с описанием атрибутов
3. Добавьте файл `gui.yaml` с описанием интерфейса

## Доступные виджеты

- **str** - строковое поле ввода
- **int** - целочисленное поле с валидацией
- **float** - поле для дробных чисел
- **text** - многострочное текстовое поле
- **list** - выпадающий список (single/multi select)
- **ip** - поле для IP адреса с маской формата
- **ip_mask** - поле для IP адреса с маской подсети
- **datetime** - выбор даты и времени
- **table** - таблица данных
- **button** - кнопка с различными действиями

### Параметры виджета button

- **description** - текст кнопки
- **info** - подсказка под кнопкой
- **command** - команда для выполнения
- **url** - URL для перехода (приоритет над command)
- **dialog** - диалог подтверждения
- **icon** - путь к иконке в папке `templates/icons/` или FontAwesome класс
- **size** - высота кнопки в пикселях (по умолчанию стандартная высота Bootstrap)
- **readonly** - только для чтения
- **output_attrs** - атрибуты для передачи в команде

## API Endpoints

- `GET /api/config` - полная конфигурация системы
- `GET /api/pages` - список доступных страниц
- `GET /api/page/<name>` - конфигурация конкретной страницы
- `GET /api/attrs` - все атрибуты системы
- `POST /api/execute` - выполнение команд
- `POST /api/reload` - перезагрузка конфигурации
- `GET /api/debug/*` - отладочные эндпоинты

## Особенности

- Горячая перезагрузка конфигурации без перезапуска
- Валидация данных на стороне клиента
- Модульная архитектура виджетов
- Полная кастомизация через YAML
- Встроенная панель отладки
